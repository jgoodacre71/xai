# Guidance Maintainer

## Purpose

Turn repeated pain into checked-in process.

## Workflow

1. Identify the repeated mistake or missing instruction.
2. Decide the right home for the guidance:
   - AGENTS.md for always-on repo rules
   - docs/ for detailed source of truth
   - a skill for repeatable workflows
   - a task template or runbook for process changes
3. Make the smallest durable documentation change that would have prevented the issue.
4. Briefly note why the new guidance exists.
