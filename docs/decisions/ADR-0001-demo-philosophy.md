# ADR-0001: Demo-first repository

## Status
Accepted

## Decision
This repository will be organised around a small number of coherent demos rather than a broad method zoo.

## Rationale
A smaller number of strong end-to-end stories is more compelling and maintainable than many shallow comparisons.

## Consequences
- fewer models in the first release;
- clearer notebooks;
- easier testing and maintenance;
- stronger visual and narrative identity.
