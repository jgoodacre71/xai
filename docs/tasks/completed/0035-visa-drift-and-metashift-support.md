# 0035: VisA Drift and MetaShift Support

## Status
Complete

## Owner
Codex thread

## Why
The repo had second-wave dataset support for VisA, but it was still invisible
in the demos. Separately, the remaining dataset-expansion TODO from the spec
was to support MetaShift or Spawrious for natural-context shortcut work. This
task addressed both gaps in one pass.

## Scope
- Extend Demo 08 so it can render optional VisA anomaly-drift sections from
  prepared VisA manifests.
- Add local MetaShift support for the published cat-vs-dog indoor/outdoor
  subpopulation-shift split.
- Extend the dataset CLI, tests, docs, and task memory for both changes.

## Out of scope
- A full MetaShift-backed Demo 01 report in the same task.
- Adding Spawrious in parallel.
- Adding ProtoPNet or another comparator in this pass.

## Deliverables
- Updated `src/xai_demo_suite/reports/explanation_drift.py`
- Updated demo CLI and suite plumbing
- `src/xai_demo_suite/data/downloaders/metashift.py`
- Focused tests for VisA drift and MetaShift support
- Updated docs, registry metadata, TODOs, and task memory

## Constraints
- Keep fresh-clone behaviour intact when no external VisA or MetaShift data is
  prepared.
- Be explicit that MetaShift preparation depends on upstream Visual Genome / GQA
  assets and scripts.
- Keep raw and generated datasets uncommitted.
- Use UK English in docs and user-facing text.

## Validation plan
1. `./.venv/bin/ruff check src tests`
2. `./.venv/bin/mypy src`
3. `./.venv/bin/pytest tests/unit/test_explanation_drift_report.py tests/unit/test_metashift_downloader.py -q`
4. `./.venv/bin/xai-demo-data list`
5. `./.venv/bin/xai-demo-data fetch metashift --category subpopulation_shift_cat_dog_indoor_outdoor --dry-run`
6. `./.venv/bin/xai-demo-report explanation-drift`

## Risks
- The published MetaShift generation scripts can change folder names or group
  pickle contents, so the adapter needs to fail clearly when the expected split
  layout is absent.
- Demo 08 can become bloated if additional anomaly sections are not kept tight.

## Decision log
### 2026-04-16
- Decision: Use VisA in Demo 08 rather than creating a separate standalone
  report first.
- Reason: That makes the second-wave dataset work visible while preserving the
  existing four-pillar demo structure.

### 2026-04-16
- Decision: Support MetaShift via the published cat-vs-dog indoor/outdoor split
  generated by the upstream scripts, rather than trying to auto-build the full
  dataset.
- Reason: That is the most directly relevant natural-context shortcut path for
  Pillar A and keeps the local adapter honest about upstream dependencies.
